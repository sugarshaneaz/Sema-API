Add onboarding questions for all 14 niche packs. Each niche's onboardingFields array should contain the questions below. The frontend renders these dynamically using field types: text, select, multi-select, boolean, textarea, number. The API endpoint GET /api/niches/:nicheKey returns a niche pack with an onboardingFields array. Each field object has: key, type, label, help (optional), required (boolean), options (array of {value, label} for select/multi-select), placeholder (optional), defaultValue (optional).

Here are the questions per niche:

cuisine_type (text) — "What type of cuisine do you serve?"

special_requests (boolean) — "Do you accept special requests (spice level / allergies / custom orders)?"

Implementation notes:

All fields should have required: true except delivery_areas fields (which are conditional)
Boolean fields render as Yes/No toggles on the frontend
Multi-select fields with a max should include max in the field metadata if supported
The frontend fetches these via GET /api/niches/:nicheKey and reads the onboardingFields array
Answers are saved via POST /api/business/:businessId/onboarding-answers with { answers: { key: value } }
