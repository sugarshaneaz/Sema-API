You are working in Sema-API (Node.js 20 + Express + TypeScript + Prisma + Postgres).

Goal: Implement WhatsApp Embedded Signup onboarding so users can connect WhatsApp from inside the Expo app without going to Business Manager.

Use Meta’s official Embedded Signup approach (Facebook JavaScript SDK hosted on a web page).
We will host a small web page at GET /connect/whatsapp that runs the embedded signup flow and then calls our API to save the connection.

Requirements:

A) Database
1) Ensure we have a WhatsAppConnection model/table that stores:
   - wabaId
   - phoneNumberId
   - displayPhoneNumber (optional)
   - businessName (optional)
   - accessToken (encrypted or masked at minimum)
   - enabled, mode (OFF|REVIEW|AUTO), pausedUntil, lastInboundAt, lastOutboundAt
   If model already exists, keep it and add missing fields via Prisma migration.

B) Web onboarding page (server-rendered HTML)
2) Add GET /connect/whatsapp
   - Returns a clean HTML page (no React required) with:
     - A single button: “Continue with Facebook”
     - Loads the Facebook JavaScript SDK
     - Starts the WhatsApp Embedded Signup flow
   - This page must use ONLY backend env vars for:
     - META_APP_ID
     - META_CONFIG_ID (Facebook Login for Business configuration for/block for Embedded Signup)
     - META_REDIRECT_URI (this same backend route or a callback route)
   - After the embedded signup completes, extract from the response the key identifiers
     (at minimum: wabaId and phoneNumberId, and any displayPhoneNumber if available)
   - Then POST to our backend endpoint (below) to store the connection in Postgres.

C) Backend endpoints to finalize + store
3) Add POST /api/whatsapp/embedded-signup/complete
   - Protected by an onboarding session token OR a signed nonce generated by backend before opening the page.
   - Input: { wabaId, phoneNumberId, displayPhoneNumber?, businessName?, shortLivedToken? or authCode? }
   - If we receive an auth code or short-lived token, exchange it server-side for the correct access token.
   - Save/update WhatsAppConnection record for this user/business.
   - Then ensure we subscribe the app to the WABA webhooks via Graph API: POST /{waba-id}/subscribed_apps using the proper token/permissions.
   - Return { ok: true, connection }.

4) Add GET /api/whatsapp/embedded-signup/status
   - Returns whether the current user has a connected WhatsApp number and its mode.

D) Webhook subscription
5) Implement/verify webhook subscription management:
   - Use Graph API “Subscribed Apps” edge: POST /{wabaId}/subscribed_apps (subscribe) and GET for verification.
   - Add logs and errors with actionable messages.

E) Security
6) Do NOT expose secrets to the Expo frontend.
7) The /connect/whatsapp page must NOT leak tokens in query strings.
8) Add a DEBUG_KEY protected endpoint to view onboarding attempts for troubleshooting.

Deliverables:
- Code changes, Prisma migration, and clear README section:
  “How to set up META_APP_ID / META_CONFIG_ID / WHATSAPP_VERIFY_TOKEN / Redirect URIs”
- Add curl examples to verify the connection exists and subscription succeeded.
